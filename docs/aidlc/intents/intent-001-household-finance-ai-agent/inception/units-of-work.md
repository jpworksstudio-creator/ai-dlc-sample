# ユニットオブワーク

## ユニット判断

**単一ユニット（Single Unit）**

## 判断理由

- 小規模グリーンフィールド MVP で、父1名が使う単一 Web アプリ
- 全ストーリー（S-1〜S-8）が同一のユーザー価値（家計把握・AI 相談）に収まる
- デプロイ単位は1つの Web アプリケーションで十分
- チーム所有・運用責任も単一プロダクトとして扱える
- コンポーネント間の依存は論理分割であり、独立デプロイの必要はない

## ユニット U-1: 家計 AI エージェント Web アプリ

| フィールド | 値 |
|---|---|
| 名前 | 家計 AI エージェント Web アプリ |
| 種別 | module（単一フロントエンド中心アプリ） |
| ユーザー価値 | 家計エクスポートの取り込みと AI による家計相談・改善支援 |
| 責務 | データ取り込み、ローカル保存、AI Q&A、カテゴリ補正表示、改善案・良かった点の提示、データ削除 |
