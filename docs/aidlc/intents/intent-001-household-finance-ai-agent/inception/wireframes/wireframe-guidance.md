# ワイヤーフレームガイダンス

## 標準フォーマット

HTML をデフォルトとする。低忠実度・モバイルファースト（320px 以上）。

## デザインシステム

- 既存デザインシステム: なし（MVP はワイヤーフレームレベル）
- 情報密度: スマホ向けに1画面1主要タスク

## 画面ガイダンス

### SCR-1: ホーム

#### レイアウト

- ヘッダー → 状態サマリー → CTA → クイック質問 → フッターナビ

#### 操作

- 入力: なし
- ボタン: データ取り込み、AI に相談
- ナビ: フッター4タブ

#### 状態

- Loading: データ読み込み中スピナー
- Empty: 「データを取り込んでください」+ 取り込み CTA
- Error: データ読み込み失敗メッセージ
- Success: 件数・最終取り込み日表示

#### レスポンシブ

- Mobile: 単一カラム、CTA 全幅
- Desktop: 最大幅 480px 中央

#### アクセシビリティ

- ボタンに accessible name
- フォーカス順: 上から下、フッターナビ最後

#### トレーサビリティ

対応ストーリー: S-1, S-2, S-3

---

### SCR-2: データ取り込み

#### レイアウト

- ヘッダー → アプリ選択ラジオ → ファイル選択 → 取り込みボタン → 結果表示

#### 操作

- 入力: アプリ種別（radio）、ファイル（file）
- ボタン: 取り込む、戻る
- ナビ: 戻る → SCR-1

#### 状態

- Loading: 「取り込み中…」
- Empty: ファイル未選択
- Error: 形式不正・読み込み失敗
- Success: 件数表示 + ホームへ
- Validation Error: アプリ未選択 / ファイル未選択

#### レスポンシブ

- Mobile: フォーム要素全幅

#### アクセシビリティ

- radio / file input に label 関連付け
- Error はフィールド直下に表示

#### トレーサビリティ

対応ストーリー: S-1, S-2

---

### SCR-3: AI チャット

#### レイアウト

- ヘッダー → 会話エリア（スクロール）→ 固定入力欄

#### 操作

- 入力: 質問テキスト（textarea）
- ボタン: 送信
- ナビ: サンプル質問チップ

#### 状態

- Loading: 応答待ちインジケータ（30秒以内想定）
- Empty: データ未取り込み時は SCR-1 へ誘導
- Error: AI 応答失敗
- Success: 応答表示（金額・明細・改善案・良かった点）

#### レスポンシブ

- Mobile: 入力欄画面下部固定、会話エリア可変高

#### アクセシビリティ

- textarea に label（視覚的に隣接）
- 送信ボタンに「質問を送信」ラベル
- `prefers-reduced-motion`: Loading アニメーション省略可

#### トレーサシビリティ

対応ストーリー: S-3, S-4, S-5, S-6, S-7

---

### SCR-4: 設定

#### レイアウト

- ヘッダー → 説明テキスト → データ件数 → 削除ボタン

#### 操作

- ボタン: データを削除（破壊的、確認ダイアログ必須）
- ナビ: 戻る

#### 状態

- Success: 削除完了メッセージ
- Error: 削除失敗
- 破壊的操作: 確認ダイアログ + キャンセル導線

#### レスポンシブ

- Mobile: 削除ボタン全幅、警告色

#### アクセシビリティ

- 削除ボタンに明確なラベル
- 確認ダイアログはフォーカストラップ

#### トレーサビリティ

対応ストーリー: S-8
