# 要件

## インテント概要

4人家族の**父**向け Web 家計 AI エージェント MVP。Zaim / Moneytree のエクスポートを端末内に取り込み、カテゴリ誤りや判別しづらい明細を補正した上で、自然言語で支出把握・無駄遣い指摘・改善提案・良かった点の評価を受けられる。

## 機能要件

- FR-1: ユーザーは Zaim のエクスポートファイルを Web アプリに取り込める
- FR-2: ユーザーは Moneytree のエクスポートファイルを Web アプリに取り込める
- FR-3: 取り込んだ家計データはブラウザローカルにのみ保存され、サーバーへ送信されない
- FR-4: ユーザーは自然言語で AI エージェントに家計について質問できる
- FR-5: AI エージェントはユーザーの質問に対してテキスト応答を返す
- FR-6: AI エージェントはカテゴリ別（食費・日用品・固定費を含む）の支出額を回答できる
- FR-7: AI エージェントは取引明細をユーザーが理解しやすい形で提示できる
- FR-8: AI エージェントは無駄遣いの指摘と家計の改善案を提案できる
- FR-9: AI エージェントは家計の良かった点を評価し、改善案とセットで返せる
- FR-10: ユーザーは取り込み済みの家計データを削除できる

## 非機能要件

- NFR-1: スマホブラウザ（画面幅 320px 以上）で FR-1〜FR-10 の主要操作が可能である
- NFR-2: 代表質問5種類のうち4種類以上について、30秒以内に応答が返る（SM-1）
- NFR-3: 食費・日用品・固定費の3カテゴリについて、手動補正後の金額と±10%以内で回答する（SM-2）
- NFR-4: 家計相談10回中8回以上で、改善案1件以上と良かった点1件以上を含む応答を返す（SM-3）
- NFR-5: 家計データは端末外（サーバー・外部 API への永続送信）に保存しない（D-7）

## 事実

- 主利用者は父、想定世帯は4人家族（父・母・2歳の子）
- 対応家計簿アプリは Zaim と Moneytree
- プラットフォームは Web アプリ（スマホブラウザ）
- プロダクト仮説は承認済み（2026-06-07）
- ワークフローは承認済み（2026-06-07）

## 仮説

- H-1: エクスポートデータの金額・カテゴリ誤りを補正し、意図したカテゴリで支出把握できる
- H-2: 判別しづらい取引内容を理解できる形に整理できる
- H-3: エクスポートだけで MVP に必要な取引データを十分に取得できる
- H-5: 自然言語 Q&A が手動確認より負担が低い
- H-6: 改善案と良かった点の評価をセット提供すると継続利用しやすい

## 前提（Assumptions）

- A-1: Zaim と Moneytree は CSV 等のファイルエクスポートを提供する
- A-2: 父がエクスポート操作（家計簿アプリ側）を実行できる
- A-3: MVP 検証は父1名による利用を前提とする
- A-4: AI エージェントの応答生成には外部 LLM API を利用する。送信は質問文 + 関連取引要約のみ（D-17）。永続保存は端末内のみ（D-7）
