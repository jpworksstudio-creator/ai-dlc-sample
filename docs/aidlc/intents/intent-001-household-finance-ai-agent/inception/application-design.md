# アプリケーション設計

技術非依存の論理設計。フレームワーク・クラウド・ライブラリ・DB・プロトコルの具体選定は人間の承認後に行う。

## コンポーネント

| コンポーネント | 目的 | 責務 | 状態 | 所有データ |
|---|---|---|---|---|
| UI シェル | 画面遷移とナビゲーション | SCR-1〜4 の表示、フッターナビ、状態メッセージ | Stateless | なし |
| 取り込みモジュール | エクスポートファイルの解析 | Zaim/Moneytree 形式の検証・パース・正規化 | Stateless | なし |
| ローカルデータストア | 端末内永続化 | 家計データの保存・読み出し・削除（ブラウザローカル） | Stateful | HouseholdDataset, Transaction |
| 取引正規化エンジン | カテゴリ・明細の補正 | 誤分類の補正、判別困難な明細の説明生成 | Stateless | なし |
| AI エージェントゲートウェイ | 自然言語 Q&A | 質問とコンテキストの送信、応答の受信（外部 AI サービス） | Stateless | なし |
| チャットプレゼンター | 応答の構造化表示 | 金額・明細・改善案・良かった点の整形 | Stateless | なし |
| 設定マネージャ | データライフサイクル | 保存方針の説明、全データ削除 | Stateless | なし |

| 操作 | 目的 | 入力 | 出力 | エラー |
|---|---|---|---|---|
| importExport | エクスポート取り込み | ファイル、アプリ種別 | ImportResult | 形式不正、パース失敗 |

## 横断的関心事

- **認証**: MVP では不要（父1名・端末内利用）
- **認可**: MVP では不要（単一ユーザー）
- **入力検証**: ファイル形式・アプリ未選択・空質問の Validation Error
- **ログ**: クライアント側のデバッグログのみ（家計データの永続送信なし）
- **プライバシー**: 家計データの永続保存は端末内のみ（D-7）。AI 応答時のみ質問文 + 関連取引要約を外部 API に一時送信（D-17）
- **エラーハンドリング**: 取り込み失敗、AI タイムアウト（30秒）、AI 応答失敗を UI に表示

## 外部依存

- 家計簿アプリ（Zaim / Moneytree）のエクスポート機能（ユーザー操作）
- 外部 AI サービス（自然言語応答生成）— 具体プロバイダ未選定（Q-2）。送信データは質問 + 関連取引要約（D-17）

## 制約

- 家計データの永続保存は端末内のみ（D-7, NFR-5）
- フレームワーク・クラウド・ライブラリ・DB の選定は本ドキュメントでは行わない
- Construction・インフラ構築は対象外
