# Application Design Checklist

## 確認項目

- [ ] Components が明示されている
- [ ] Responsibilities が明示されている
- [ ] Dependencies が説明されている
- [ ] Services が Story と対応している
- [ ] Cross-cutting Concerns がある
- [ ] Data Model が Wireframe の表示 / 入力データを支えられる
- [ ] API Overview が必要十分である
- [ ] External Dependencies が明示されている、または None と記録されている
- [ ] 認証、入力検証、ログ、エラーハンドリングが整理されている
- [ ] Framework、Cloud Service、ライブラリ、細かい実装方式へ早期に寄せすぎていない

## 未解決なら停止する条件

- Application Design が未承認である
- Wireframe のデータ需要を支えられない
- Story に対応しない設計要素がある
- 実装技術の未承認決定が混ざっている
