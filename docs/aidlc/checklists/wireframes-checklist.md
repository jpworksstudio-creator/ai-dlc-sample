# Wireframes Checklist

## 確認項目

- [ ] `screen-data-map.md` がある
- [ ] `screen-structure.md` がある
- [ ] `wireframe-guidance.md` がある
- [ ] `screens/<screen-name>.html` がある
- [ ] 各画面が User Story と対応している
- [ ] 画面遷移が説明されている
- [ ] 表示データと入力データが明示されている
- [ ] ユーザー操作が明示されている
- [ ] Loading / Empty / Error / Success / Validation Error / 権限不足が確認できる
- [ ] Mobile 幅での表示方針がある
- [ ] 入力欄とボタンが確認できる
- [ ] アクセシビリティー観点がある
- [ ] ナビゲーションと情報密度が確認されている
- [ ] 既存デザインシステムの有無が確認されている
- [ ] 要件に存在しない機能を勝手に追加していない

## Web Design Guidelines 由来の観点

- [ ] Icon-only controls には accessible name がある
- [ ] Form controls には label がある
- [ ] Error は field 近くに表示される
- [ ] Focus order と keyboard 操作を妨げていない
- [ ] `prefers-reduced-motion` が必要な動きは記録されている
- [ ] Empty state と長いテキストを想定している
- [ ] Destructive action は確認または取り消し導線を想定している

## 未解決なら停止する条件

- Wireframes が未承認である
- UI Story に対応する画面がない
- 画面が要件にない機能を追加している
- 入力、状態、エラー、Mobile 表示が確認できない
